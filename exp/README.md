# 🧠 MemRL: 面向大语言模型的通用记忆强化学习框架
**(Memory-Augmented Reinforcement Learning for Generalist LLM Agents)**

MemRL 是一个专为大语言模型（LLM）智能体设计的连续学习（Continual Learning）框架。本框架摒弃了传统的神经网络权重微调范式，通过构建**“外部情节记忆库 (Episodic Memory)”**与**“通用物理势能奖励引擎”**，使参数冻结（Frozen）的大模型在与环境的动态交互中，实现自主的经验提炼、策略进化与价值优化。

## 📂 核心架构与模块化说明

本项目采用高内聚、低耦合的模块化设计，将决策核心（LLM）、外部存储（Memory）、环境交互接口（Environment）与价值优化器（RL Optimizer）相互隔离，确保系统可扩展性。

### 🚀 1. 系统主控模块
* **`run_episode.py` (生命周期调度引擎)**
  * **作用**：整个系统的中央控制台。负责模块初始化、动态上下文构建（融合环境观测状态与历史动作滑动窗口）、执行决策步，并触发全局信用分配与价值结算流程。

### 🧩 2. 决策与评估模块 (`modules/`)
* **`llm_core.py` (执行器与反思器)**
  * **作用**：封装 LLM 交互链路，包含动作空间生成（`generate_action`）与经验归纳反思（`summarize_experience`）。
  * **核心机制**：引入**“强制思维链 (CoT)”**机制约束输出方差，要求模型在生成动作前，必须将抽象经验映射为当前环境的具象实体，从而抑制概率性输出导致的轨迹震荡与逻辑跃迁。同时，强制生成跨域（Domain-Agnostic）的抽象策略法则，避免简单的行为复现。
* **`rl_optimizer.py` (通用势能审计引擎)**
  * **作用**：脱离具体底层代码，通过**状态差 ($\Delta S$)** 和 **信息熵增量** 来量化评估智能体的决策效能。
  * **核心机制**：设计动作成本惩罚（Cost Budget）机制抑制死锁与冗余。引入**基于最小化约束的信用分配策略（Min-Bottleneck Credit Assignment）**，将未来轨迹中的次优灾难性动作惩罚反向传播，精准计算状态-动作节点的真实势能 ($G_t$)。

### 💾 3. 记忆检索与存储模块 (`modules/`)
* **`retriever.py` (策略检索器)**
  * **作用**：在新任务输入时，基于当前意图的特征表征与记忆库进行高维向量空间检索，执行策略召回。
* **`memory.py` (情节记忆管理)**
  * **作用**：管理情节记忆（Episodic Memory）的节点聚合、特征存储及硬盘持久化（JSON 序列化）。
* **`encoder.py` (意图编码器)**
  * **作用**：将自然语言形式的任务指令（Task）映射为高维特征向量 ($z_t$)，支撑检索器完成基于余弦相似度的语义匹配。

### 🌍 4. 环境与基建模块
* **`env_wrapper.py` (环境交互中间件)**
  * **作用**：作为系统与外部环境的解析适配器。目前适配 ALFWorld 环境，负责将底层引擎的原始回馈转化为规范化的文本观测状态（Observation）及合法动作空间（Valid Actions）。
* **`requirements.txt`**
  * **作用**：项目依赖配置文件。

---

## 🎛️ 核心超参数配置 (Hyperparameters)

### 🎯 记忆检索双阈值机制 (`retriever.py` / config)
* **`retrieval_threshold` (默认 0.70)**: **检索召回阈值（只读模式）**。定义策略泛化的相似度下界。
* **`merge_threshold` (默认 0.99)**: **节点融合阈值（读写模式）**。决定记忆库特征的正交性。仅当检索相似度超过此极高阈值时，RL 优化引擎方可对既有记忆节点的 Q 值进行原址覆盖；否则，系统将开辟独立的记忆表征节点，防止特征污染。

# 🧠 MemRL 核心机制：泛化强化学习与价值分配引擎
**(Universal Reward Shaping & High-Water Mark Credit Assignment)**



传统强化学习中的奖励函数（Reward Function）往往高度依赖特定环境的硬编码，且其时序差分（TD）更新强依赖概率期望。然而，在 LLM 驱动的智能体架构中，情节记忆作为静态的高阶逻辑先验，若采用传统的误差衰减公式，易引发“高质量策略价值过度衰减”的问题。

为此，本框架构建了一套适用于开放文本域的通用评估法则。

## 一、 通用物理审计引擎 (Universal Physics Engine)

智能体在交互过程中的单步动作评估，不再依赖特定环境的分数标定，而是通过**状态差 ($\Delta S$)** 和 **信息熵** 进行泛化的物理核算。

每一步的单节点净收益公式定义为：
$$R_t = A_t - C_t$$

### 1. 资产增量评估 ($A_t$ : Asset)
代表该动作产生的**正向状态转移探索价值**。
* **💎 终止状态达成 ($A_t = 10.0$)**：环境触发 `is_success=True` 标志，任务目标成功闭环。
* **💰 目标约束突破 ($A_t = \text{step\_reward} \times 2.0$)**：底层环境反馈局部正向奖励（如完成关键子任务）。
* **👁️ 视野与状态扩展 ($A_t = 0.5$)**：即便无显式得分，只要该动作驱动了环境状态的实质性拓扑转移 ($\text{obs} \neq \text{next\_obs}$)，系统即认可其状态空间探索（State Space Exploration）价值。

### 2. 动态噪音审计 ($C_t$ : Cost Budget)
代表该动作引发的**系统熵增惩罚与算力消耗成本**。设定基础成本 `base_cost = 0.1`，旨在抑制低效的探索轨迹。
* **❌ 状态停滞与违规约束 ($C_t = 1.0$)**：动作导致环境状态无转移 ($\Delta S = 0$) 或触发非法指令约束。由于该步决策未能产生有效信息增益，系统施加最大惩罚权重。
* **🔄 循环死锁检测 ($C_t = 0.8$)**：当前动作在短期观测滑动窗口（如最近 3 步）中高频复现。此现象表征策略生成陷入局部死锁，系统通过施加高额惩罚阻断自回归循环。
* **🛡️ 动作合法转移 ($C_t = 0.1$)**：状态正常推进，仅扣除基础时间步衰减成本。

---

## 二、 最小路径约束信用分配 (Min-Path Bottleneck Credit Assignment)



传统的贝尔曼更新（Bellman Update）采用未来奖励期望的累加衰减，在长序列文本推理中，这种方式极易掩盖因局部逻辑断裂而导致的全局失败。

本系统引入**基于路径最小约束的信用分配方法**计算全局节点势能 ($G_t$)：
$$G_t = R_t + \gamma \times \min(\{R_{t+1}, \dots, R_T\})$$
*(其中 $\gamma = 0.95$ 为折扣因子)*

**算法核心逻辑**：
某一决策节点的真实长期价值，不仅由局部即时收益 ($R_t$) 决定，更受制于其所引发的后续状态链条中的“最差次优约束”。若轨迹在远期触发灾难性惩罚（例如 $R_5 = -5.0$），该极小值将通过 $\min$ 函数被反向传播，对该因果链上的所有前置节点进行强力的价值修正。

---

## 三、 高水位线价值更新机制 (High-Water Mark Value Update)

针对静态记忆库结构的特性，本框架重构了传统的价值评估范式。

### 1. 传统 TD 学习的局限性
采用标准公式 $Q = Q + \alpha \times (G_t - Q)$ 时，即使智能体借用高权重的先验记忆（如 $Q = 5.0$）且单轮轨迹表现最优，其受限于基础步数成本 ($C_t = 0.1$) 计算得出的实际累积回报 $G_t$ 往往低于该先验。这将触发 TD 算法的负向误差更新，导致最优策略的表征价值被异常稀释。

### 2. 理论上限记录策略
设定记忆节点的长期价值等同于该策略所能激发的**最大潜能上限**：
$$Q_{new} = \max(Q_{old}, G_t)$$

* **[最优路径刷新]**：若当前轮次智能体探索出效率更优的状态转移路径 ($G_t > Q_{old}$)，系统执行价值覆盖，永久提升该经验的权重。
* **[策略价值保护]**：若当前探索因随机噪音或生成偏差导致低效循环 ($G_t < Q_{old}$)，系统拒绝向下修正价值，从而防止劣质评估轨迹污染高质量的先验经验库。

### 3. 稳健性截断机制 (Victory Guarantee)
$$G_t = \max(G_t, 0.5) \quad \text{if } \text{is\_success} == \text{True} \text{ and } G_t < 0$$
若当前回合最终达成目标状态，但由于早期的探索震荡导致核算的全局势能 $G_t < 0$，系统利用下界截断机制将其强制修正为 0.5。以此确保能够导向最终成功态的有效探索序列，不会被错误地标记为负向预期价值。

### 🧠 模型生成约束 (`llm_core.py`)
* **`temperature` (默认 0.1)**: 策略生成方差控制。在动作生成阶段限制大模型的采样温度参数，旨在保证同一观测状态下策略输出的确定性与一致性。

## 📈 训练输出与数据追踪

* **评估数据持久化 (`logs/training_metrics.csv`)**：在每回合结束时，系统将自动序列化记录执行轮次（Episode）、任务分类（Task_Type）、终止状态（Success）与累计回报（Total_Reward）。该数据流支持与可视化框架无缝集成，用于离线生成模型收敛曲线及泛化效能图表。
